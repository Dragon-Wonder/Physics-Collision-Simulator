22 # Running the program
23 
24 In order to run the program properly you will need to download SDL2-2.0.3 or higher AND SDL2-image (these are two different libraries).
25 
26 A link for SDL2 can be found [here] (https://www.libsdl.org/download-2.0.php).
27 
28 A link for SDL2 image can be found [here](https://www.libsdl.org/projects/SDL_image/) .
29 
30 You will want to download the Runtime Binaries (if you have windows I recommend 32 bit over 64 bit; even if you have a 64 bit computer).
31 
32 When all is said and done you should have the following files in your folder. (Also note that you should be able to put the dll files in your C drive).
33 
34 ```
35 FolderName\
36 FolderName\Cannon.exe
37 FolderName\SDL2.dll
38 FolderName\SDL2_image.dll
39 ```

43 # Using the Program
44 
45 The first time you run the program a "Config.ini" will be created it should look something like below:
46 
47 ```
48 Config File for the Cannon.exe
49 1.0.0
50 Screen Width: 640
51 Screen Height: 480
52 Log Ball's path: 0
53 Enable Drag Mode (experimental): 0
54 Draw Ball path on screen: 0
55 Collision Method: 1
56 Number of points to track: 5
57 Delay between recording points: 25
58 ```
59 
60 You can use this to change things like the screen height and width, and other stuff.
61 
62 "Log Ball's Path" (if enabled with a 1 instead of 0), will cause a a file called "logfile.log", which will contain the placement of every ball on every update (it gets quite big as you can guess).
63 
64 "Enable drag Mode" (if enabled with a 1 instead of 0), will cause the cannonballs to be effected by drag. It is listed as experimental because I based the equations used off of several things I found
65 online and I cannot verify how accurate it is.
66 
67 "Draw Ball path on Screen" (if enabled with a 1 instead of 0), will cause small pixels to trail after each ball.
68 
69 "Collision Method" you can change how the balls will interact with each other when they collide based on the number in the table below.
70 
71 | Name | Number | Explanation |
72 |-------------------------|--------|------------------------------------------------------------------------------------------------------------|
73 | CollideElastic | 0 | This is normal collision in a perfect world without friction, they hit and bounce off, no energy is lost. |
74 | CollideInelastic | 1 | This is the real life situation. Balls collide and, bounce off of each other losing some energy. |
75 | CollidePerfectInelastic | 2 | In this method the balls collide and then stick together. |
76 | CollideNone | 3 | The balls do not collide and pass through each other. |
77 
78 
79 "Number of points to track" is the number of past points that will be kept track of if "Draw Ball Path on screen" is enabled.
80 
81 "Delay between recording points" is the delay (in ticks) between everytime it records a point, if "Draw Ball Path on screen" is enabled.
82 A smaller number means that the points are recorded much faster.
83 
84 Once you start the program you will be greeted with a beautiful black screen. To create a new cannonball you click (and hold down the button)
85 you can now move your move around to change the angle that the new cannonball will be fired at; as well as the longer the line the faster it will go.
86 When you are happy with the angle and speed simply release the mouse button and it will be created at the point where you clicked the mouse button down.
87 
88 The longer you hold down the mouse button the larger the radius of the ball will be and as a result, the more mass the ball will have. You can get an
89 idea of how much mass a ball has based on how transparent it is, the more transparent it is the lighter its mass.
90 
91 You can also press the following keys to yield effects
92 
93 ```
94 K = All Balls will be "killed".
95 R = All motion will be stopped.
96 Q / ESC = Program will exit.
97 F = Every so often a randomly generated cannonball will be created
98 ```
